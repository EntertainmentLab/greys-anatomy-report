## **Additional Analysis Notes**

### **Dynamic Classes (Generated at Runtime)**
These classes are created dynamically by D3 or JavaScript:
- point-Control, point-Treatment, point-Handoff (D3 point classes)
- bar-* (D3 bar classes for different categories)
- Various condition-specific classes based on data

### **Pseudo-Classes and States**
The following classes are used with pseudo-classes in CSS:
- :hover states for buttons, links, interactive elements
- :focus states for form controls and interactive elements
- :active states for buttons and toggles
- :before and :after pseudo-elements for icons and decorations
- :disabled states for buttons
- :checked states for checkboxes and toggles

### **Media Query Variants**
Many classes have responsive variants that apply at different breakpoints, but use the same base class names listed above.

### **CSS Nested Selectors**
Many classes are used in nested selectors like:
- .parent .child combinations
- .class1.class2 combinations
- .class:pseudo-class combinations

## **Summary**
This comprehensive list represents **over 400 unique CSS classes** that are actively referenced and used throughout the application, including:
- Direct className usage in JSX components
- CSS selectors and their variants (hover, active, pseudo-elements)
- Dynamically generated classes in D3 visualizations
- Nested and combined selectors
- Modifier classes and state variations
- All responsive and media query variants
- Classes referenced by JavaScript and D3 code

**Total Used Classes: ~400+**
